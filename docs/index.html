hydra-llm

Run local LLMs the easy way. Pick a model, download it, chat with it. No cloud, no API key, no telemetry.

No warranty. hydra-llm runs LLMs on your machine, manages Docker containers, downloads multi-gigabyte model files from third-party hosts, and exposes HTTP APIs on local ports. Install

One-liner. Installs the CLI into ~/.local/bin, builds the engine image, downloads a starter model, runs a smoke test. 
curl -fsSL https://raw.githubusercontent.com/ra-yavuz/hydra-llm/main/get.sh | bash
Do not run with sudo. The installer asks for sudo only for missing system packages (Docker, python3-yaml).
Or via apt (Debian/Ubuntu, once the apt repo card is published): 
echo "deb [trusted=yes] https://ra-yavuz.github.io/apt stable main" | \
  sudo tee /etc/apt/sources.list.d/ra-yavuz.list
sudo apt update
sudo apt install hydra-llm hydra-llm-plasma   # plasma is optional

Quick start

hydra-llm doctor          # detect your hardware
hydra-llm setup           # build engine image + starter model
hydra-llm list-online     # browse the catalog (filtered to your hardware)
hydra-llm download llama-3.1-8b
hydra-llm chat llama-3.1-8b

What you get

  • Single CLI: hydra-llm with subcommands like setup, chat, start, stop, list-online, download, doctor, uninstall, wipe.
  • Curated catalog of community-quantized GGUFs that download anonymously.
  • Two engine images (Vulkan, CPU) built locally on first setup; auto-fallback if the preferred image fails.
  • Persona files: drop a markdown file with optional YAML front matter into ~/.config/hydra-llm/personas/ and --persona <name> picks it up.
  • Persistent chat sessions in ~/.local/state/hydra-llm/sessions/; resume with --session <name>.
  • OpenAI-compatible HTTP API exposed by every running model. hydra-llm api <id> prints a copy-paste curl command.
  • Optional KDE Plasma 6 widget (hydra-llm-plasma) with a HAL-eye animated indicator and inline log console.

Privacy

  • No telemetry, no analytics, no auto-update calls.
  • All inference runs locally in Docker.
  • Chat sessions stay on your disk; the CLI never uploads them.
  • Model downloads work without a Hugging Face account. HF_TOKEN is supported but never required.
